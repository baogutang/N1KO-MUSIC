## 修复全屏播放器阴影闪动问题

**文件**：`frontend/src/components/player/FullscreenPlayer.tsx`

**问题根因**：
1. 背景渐变颜色 (`bgColors`) 在 `getCachedColors` 异步返回后直接 setState，没有过渡动画，导致颜色突变
2. `getCachedColors` 在 `coverLoaded=false` 时就被触发（由 `coverUrl` 驱动），提前取到旧/临时颜色，切歌时颜色闪跳
3. 模糊背景层 `<img>` 使用 `key={currentSong.id}` 强制重建 DOM，切歌时旧元素直接消失，新元素 opacity 从 0 开始，与背景颜色变化不同步，产生「阴影闪烁」

**修复方案**：
1. 新增 `displayBgColors` state，`bgColors` 更新后用 `requestAnimationFrame` 延一帧再同步，确保 CSS transition 有时间接手
2. 根 div 的 `background` 改用 `displayBgColors`，并加 `transition: 'background 1.2s ease'`
3. `getCachedColors` 的触发条件改为 `coverLoaded=true` 后才执行，避免提前取色
4. 模糊背景层移除 `key={currentSong.id}`（不再强制重建 DOM），改用 `willChange: 'opacity'` 和 `transition: 'opacity 0.8s ease'` 实现平滑淡入淡出
